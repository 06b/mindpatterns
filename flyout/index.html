Readme

Flyout is a standalone pattern, but also form the basis of many composite patterns such as tooltip, bubble help, menu and faux menu.

Use of hover and focus activated flyout is discouraged as this behaviour forces keyboard and screen reader users to navigate through overlay (thus defeating the purpose of progressive disclosure). This problem can be mitigated by use of a 'stealth' button (a button that only appears on keyboard focus).

Please read the documentation to understand how these examples were created.
